**exactOptionalPropertyTypes**  - если true, то заставляет тс считать ситуации, когда поле в объекте есть но равно `undefined` и когда вообще нет, разными.

Пример:

```ts
type System = {themeColor: 'light' | 'dark' | undefined}
```
и

```ts
type System = {themeColor?: 'light' | 'dark'}
```

Есть места в `JS`, где поведение при наличии свойства со значением `undefined` и отсутствием свойства будет разным. 

Например если я в объекте явно указал ключ со значением `undefined`, то проверка ключа через оператор `in` вернёт `true`, а если не указал ключ, то вернёт `false`. Похожая ситуация с методом `Object.keys()`.
