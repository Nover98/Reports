**noEmit** - если стоит true, то запрещает typescript при запуске команды tsc (компилятора) создавать какие-либо выходные файлы (.js, .d.ts или .js.map).

Этот флаг превращает typescript из компилятора в контролёра. Показывать ошибки и проверять типы будет, но транспилировать файлы из ts в js не будет.










**npx** (Node Package Executable) — это инструмент, который идет в комплекте с npm. Его главная задача — запускать исполняемые файлы (бинарники) пакетов.

Когда ты пишешь npx tsc, происходит следующее:

Поиск в текущей папке: Он заглядывает в ./node_modules/.bin/tsc. Если его там нет (как в твоем случае)...

Поиск вверх по дереву (Traversing): Он идет в родительскую папку, потом в прародительскую и так далее, пока не найдет node_modules с установленным TypeScript.

Глобальный поиск: Если во всем проекте его нет, он заглянет в глобальные папки системы.

Временная установка (Download): Если пакета вообще нигде нет, npx скачает его во временную папку, запустит и удалит.
