**noUncheckedSideEffectImports** — если `true`, TypeScript начинает проверять side-effect imports строже.

Речь про импорты без привязки к переменной, которые пишутся просто ради выполнения модуля:

```ts
import "./setup";
import "some-polyfill";
```

## Поведение по умолчанию

По умолчанию значение у этого флага — `false`.

Это значит, что TypeScript может не ругаться на side-effect import даже тогда, когда модуль по этому пути не найден.

Именно поэтому этот флаг и называется `noUncheckedSideEffectImports`: без него такие импорты проверяются менее строго.

## Что меняется при `true`

Если включить `noUncheckedSideEffectImports`, TypeScript начнёт проверять, существует ли модуль, который импортируется только ради побочного эффекта.

То есть запись вроде:

```ts
import "./file-that-does-not-exist";
```

при `false` может пройти тихо, а при `true` уже даст ошибку.

## Важный нюанс

Этот флаг особенно полезен, если в проекте есть импорты вроде CSS, полифилов и файлов инициализации. Без него опечатка в side-effect import может остаться незамеченной. То есть тайпскрипт здесь может проверять наличие подключаемых модулей с любым расширением файла, но для этого обычно нужны какие-то дополнительные настройки, типа бандлера и так далее, потому что, разумеется, тайпскрипт сам по себе может работать только с ts, js, json, tsx и типа того.
