# `tsconfig.json`

Файл `tsconfig.json` помечает папку как корень TypeScript-проекта и задаёт настройки компилятора.

## Как TypeScript ищет конфиг

Если запустить `tsc` **без явного списка входных файлов**, TypeScript сначала ищет `tsconfig.json` в текущей папке, а затем поднимается вверх по родительским директориям, пока не дойдёт до корня диска.
Если `tsconfig.json` не найден, компилятор использует настройки по умолчанию.

> Если входные файлы указаны прямо в команде, например `tsc demo.ts`, то `tsconfig.json` игнорируется. Задать одновременно в одной команде и имя файла, и конфиг нельзя.

## Может ли быть несколько конфигов в одной папке

Да. В одной папке может лежать сколько угодно конфигов с разными наборами настроек для разных сценариев.

Например:

- один конфиг для браузера
- другой для Node.js
- отдельный конфиг для демо
- отдельный конфиг для сборки

## Какой конфиг используется по умолчанию

Если ничего не указывать, TypeScript пытается использовать файл с именем `tsconfig.json`.

Если нужен другой конфиг, его нужно передать через флаг `--project` или сокращённо `-p`.
Также после `-p` можно передать не конкретный файл а путь к файлу без указания имени, анпример `npx tsc -p ./src/lib/demo`. В этом случае ts будет искать файл `tsconfig.json` внтури указанной папки.

## Важно про имя файла

Если конфиг запускается через `-p`, его имя может быть почти любым. Главное, чтобы внутри был валидный конфиг TypeScript.

Примеры:

```bash
tsc -p tsconfig.bundler.json
tsc -p tsconfig.node.json
tsc -p lol.json
```

## Коротко

- без `-p` TypeScript ищет именно `tsconfig.json`
- с `-p` можно указать любой нужный конфиг
- если явно перечислить файлы в команде, `tsconfig.json` не используется
