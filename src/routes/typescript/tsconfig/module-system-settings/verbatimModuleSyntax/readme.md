**verbatimModuleSyntax** — говорит TypeScript не додумывать за разработчика, какие импорты нужно оставить в итоговом JS, а какие можно выкинуть.

Если флаг включен, TypeScript начинает относиться к импорту буквально:

- `import type` — это точно только тип, значит такой импорт будет удалён из JS
- обычный `import` — это уже настоящий импорт, и TypeScript оставит его в output как есть

То есть при `verbatimModuleSyntax: true` нужно самому явно разделять:

- что импортируется только ради типов
- а что действительно должно существовать в runtime

Почему это важно: без этого флага TypeScript может сам решить, что импорт был нужен только для типов, и удалить его из итогового JS. Иногда это удобно, а иногда из-за этого случайно пропадают side effects модули (что видно в демо примере).
