## Продвинутая возможность

С версии TypeScript 4.5 появился механизм, при котором часть стандартных `lib` можно подменять пакетами из `node_modules`.

## Зачем это вообще нужно

Иногда хочется не ждать, пока нужные типы обновятся внутри самого TypeScript.

Например:

- нужно зафиксировать другую версию DOM-типов
- нужно использовать более свежие или более совместимые типы для веб-платформы
- нужно, чтобы стандартные браузерные типы брались не из встроенного `lib.dom.d.ts`, а из внешнего пакета

## Как это работает

TypeScript умеет искать специальные пакеты вида:

- `@typescript/lib-dom`
- `@typescript/lib-webworker`
- и т.д.

Когда TypeScript решает, какие `lib`-файлы нужно подключить, он сначала проверяет, нет ли в `node_modules` специального пакета вида `@typescript/lib-*`. Например, если в `lib` указано значение `dom`, TypeScript при наличии такого пакета будет использовать типы из `node_modules/@typescript/lib-dom`.

Это позволяет:

- отвязать часть стандартных платформенных типов от версии самого TypeScript
- обновлять или фиксировать их отдельно
- точнее контролировать окружение
