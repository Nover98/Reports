# TypeScript в VS Code

VS Code под капотом использует так называемый TS Server. Это фоновый процесс Node.js, который запускается каждый раз, когда в редакторе открывается файл `.ts` или `.js`.

TS Server — «связующее звено» между редактором и компилятором TypeScript. Внутри сервера работает TypeScript Language Service — «интеллектуальное ядро». Оно:

- строит и хранит в оперативной памяти AST-дерево проекта;
- динамически анализирует изменения;
- проверяет типы и находит ошибки;
- предоставляет IntelliSense (подсказки после точки), Rename (безопасное переименование) и Go to Definition (переход к коду).

TS Server использует тот же TypeScript, который мы устанавливаем через npm, но эта версия хранится не в `node_modules`, а внутри IDE. Это иногда заметно, когда, например, сборка проекта проходит без ошибок, а в редакторе в файле всё равно что-то подчёркнуто. Или, например, в файле конфигурации подчёркивается поле, которое точно есть в актуальной версии TypeScript в проекте.

В таком случае VS Code можно переключить на версию TypeScript из `node_modules` текущего проекта:

1. Нажмите Ctrl + Shift + P.
2. Выберите TypeScript: Select TypeScript Version…
3. Выберите Use Workspace Version.

Как только вы открываете папку с проектом, VS Code начинает искать ближайший `tsconfig.json`. Если он его находит, TS Server настраивается строго по правилам этого конфига.

Если в `exclude` указан файл `eslint.config.js`, VS Code перестаёт связывать этот файл с контекстом проекта (его AST-деревом).

Если файла конфигурации нет (например, в обычном JS-проекте), VS Code использует настройки по умолчанию. Этот встроенный механизм называется Salsa: он пытается «угадать» типы, анализируя код и JSDoc-комментарии.

Также он автоматически подтягивает типы для популярных библиотек (из папки `node_modules` или через автоматическую загрузку `@types`), чтобы вы видели подсказки для Express или Lodash.
