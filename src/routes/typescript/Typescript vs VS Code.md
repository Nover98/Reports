Typescript в VS code.

VS code под капотом использует так называемый TS server. Это фоновый node.js процесс, который запускается каждый раз, когда в редакторе открывается ts или js файл. TS server является «связным» между редактором и компилятором typescript. Внутри сервера работает TypeScript Language Service - «интеллектуальное ядро». Оно строит и держит в оперативной памяти AST-дерево проекта, динамически анализирует изменения, проверяет типы и находит ошибки, предоставляет услуги IntelliSense (подсказки после точки), Rename (безопасное переименование), Go to Definition (переход к коду). TS server использует такой же typescript как и тот, что мы ставим из npm, только он хранится не в node_modules, а где-то внутри IDE. Это иногда можно заметить, когда, например, сборка проекта проходит без ошибок, а в редакторе всё равно в файле что-то подчёркивается. Или, например, в файле конфига подчёркивается поле, которое точно есть в актуальной версии ts в проекте. Возможно VS code использует другую версию TS, более старую или новую, в которой данная строка вызывает ошибку. VS code можно заставить использовать typescript из node_modules текущего проекта. 
1)Ctrl + Shift + P
2)TypeScript: Select TypeScript Version…
3)Use Workspace Version 
Как только я открываю папку с проектом, VS Code начинает искать ближайший tsconfig.json. Если он его находит, то TS Server настраивается строго по правилам этого конфига. Если я указал в exclude файл eslint.config.js, то VS Code перестанет связывать этот файл с контекстом моего проекта (его AST деревом). Если файла конфигурации нет (например, в обычном JS-проекте), VS Code использует настройки по умолчанию. Этот встроенный механизм называется Salsa. Он пытается «угадать» типы, анализируя твой код и JSDoc-комментарии.
Он автоматически подтягивает типы для популярных библиотек (из папки node_modules или через автоматическую загрузку @types), чтобы ты видел подсказки для express или lodash.
