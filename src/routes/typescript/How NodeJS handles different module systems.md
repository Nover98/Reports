Как Node.js сам работает при взаимодействии разных модульных систем без сторонних инструментов:

CommonJS (CJS): require() выполняется во время исполнения; экспорт — это значение module.exports (объект или функция). Импорт/экспорт динамический.

ESM: import/export статичны (анализируются до выполнения).

Из-за этого “одно в другое” без правил/прослойки не всегда возможно.

Что делает Node.js
Node не “транспилирует” модули автоматически как TypeScript/бандлер, но умеет запускать и CJS, и ESM и имеет встроенные правила интеропа.

Интероп (от англ. interoperability — функциональная совместимость) — это способность различных программных систем, приложений или компонентов обмениваться данными и совместно использовать информацию, даже если они созданы разными разработчиками или работают на разных платформах. 

ESM → импорт CJS (Node это разрешает)

В ESM-файле можно сделать `import x from 'cjs-пакет'`. На уровне рантайма Node оборачивает CJS-модуль в ESM-namespace примерно так:

```js
"use strict";
var __importDefault =
    (this && this.__importDefault) ||
    function (mod) {
        return mod && mod.__esModule ? mod : { default: mod };
    };
Object.defineProperty(exports, "__esModule", { value: true });
var commonJS_module_1 = __importDefault(require("./commonJS-module"));

console.log((0, commonJS_module_1.default)(10, 20));
```

default обычно будет равен module.exports целиком (то есть если CJS экспортирует функцию, default — это функция).
“Именованные импорты” `import { something } from 'cjs'` в Node могут иногда сработать, если Node сможет безопасно “угадать” свойства экспорта, но это не гарантируется и зависит от того, как устроен CJS модуль. Надёжный паттерн: 
```js
import pkg from 'cjs';
const { something } = pkg;
```

Но вот напрямую сделать `require("./module");` не получится, будет ошибка.


CJS → загрузка ESM (тут наоборот, и больнее)

require('esm-модуль') не работает (Node выдаст ошибку), потому что ESM загружается асинхронно и по другой модели.

Варианты:
из CJS использовать динамический import('esm-модуль') (возвращает Promise), или сделать ESM-обёртку/точку входа.


От чего зависит, ESM это или CJS
Node выбирает формат по:
- расширению (.mjs — ESM, .cjs — CJS),
- или по package.json → "type": "module" (тогда .js считается ESM; иначе .js обычно CJS).
